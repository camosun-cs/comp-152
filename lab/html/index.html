    About the "index.html" file

    "index.html" is a basic HTML file. All web pages that you access with a web browser are made with HTML just like this one (although real sites are usually much more complicated!).

    HTML is a "plain-text" format; this means that the file is encoded in text and you can view and edit the file by hand in a text editor. HTML is also an example of a markup language: it is a mix of regular readable text and special markup commands that give the text meaning. All the words caught between < and > characters are markup, and everything else is just regular text.

    Because of time constraints, we've provided you with this template file so that we can skip over the details of the format and get right into marking up some content. If you'd like to learn more, there are many resources online that can teach you all about the nitty-gritties!

    Tags, Elements, and Comments

    The markup commands in HTML are called tags and always start with a "<" character and end with a ">" character.

    The two main kinds of tags are opening tags (like <body>) and closing tags (like </body>). Both have text inside the "<…>" that gives the "name" of the tag, but closing tags always start with a "/" character after the initial "<". Tag names are case-insensitive, so "BODY" and "body" are equivalent. A pair of opening and closing tags with the same name create an element. Everything between the opening and closing tags of an element is that element's "content."

    For example, the code <h1>hello world</h1> defines an h1 element with the content "hello world" using a pair of opening and closing tags.

    Elements should not partially overlap one another: one element should always either fully contain or be fully contained in the other. For example, <a>one <b>two</a></b> is invalid markup, but <a>one <b>two</b></a> is fine!

    Tags beginning with "<!--" and ending with "-->" are comments. Comments are hidden from normal display, and are mainly useful for leaving yourself notes for editing. We've included some comments in the template to show you where to insert new content at each step.

    Other tags starting with "<!" are processing instructions. These are rarely used in HTML, except for the "DOCTYPE" instruction at the top of the file that declares that this is an HTML document.

    HTML headings

    Headers in HTML work similarly to how they do in MS Word. Headers create implicit sections in your document, and are a great help when scanning the content of a long page.

    There are six levels of headings in HTML, numbered from level 1 (the highest, most important heading) to 6 (the deepest, least important) which are created using the <h1>, <h2>, through to <h6> tags.

    Your web browser has some built-in styles for headers that make them stand out from the surrounding text, but you can modify their look with CSS (discussed later).

    Structural elements

    There are many structural elements in HTML (such as nav, header, section, and footer) that have no immediate visible affect in the browser. These elements add explicit structural heirarchy to your document which can be useful when adding styles and help when processing your document with other tools (like screen readers, search engines, and mobile "reader" view).

    HTML attributes

    Attributes are extra bits information that can be added to an opening tag. A single tag can have as many attributes added to it as you like, but the particular attributes that are meaningful varies for different kinds of elements.

    Attributes are added into an opening tag after the tag's name, separated by spaces. Most attributes have a name and a quoted value, joined with an equals sign. For example:

    <section id="hixie">

    creates a section tag with an id attribute with a value of "hixie". As another example,

    <img src="kitty.png" alt="A kitteh">

    creates an img tag with two attributes: a src attribute with a value of "kitty.png", and an alt attribute with a value of "A kitteh".

    Some attributes don't require a value. In these cases, you should leave out the equals sign too. For example,

    <details open>

    creates a details tag with the open attribute.

    The id attribute can be added to any element in HTML. Its value should be a unique identifier within the document, which can be used to refer to that particular element. IDs are also useful for creating in-document links. After you add the "hixie" ID to your section, you should be able to jump to this section using the "HTML is the new HTML5" link at the top of the document.

    Creating links

    The a element on its own creates an "anchor" which doesn't do a whole lot on its own. In order to link to another resource, you'll need to add an href attribute into the opening tag whose value is the URL to the linked resource. In this case, the href should have a value of "https://blog.whatwg.org/html-is-the-new-html5".

    You can link to any resource that has a URL, even if it's not an HTML page. If you click on a link to a resource that your browser cannot display, it will download that resource to your computer instead.

    You can link to sections of a document using URL "fragments". Anything after a "#" symbol in a URL is a "fragment", and it if matches the ID of an element in a web page the browser will automatically scroll to it. You can see examples of this in the nav section of your document.

    URLs can be shortened significantly by using relative addresses. To access a resource in the same folder as the current page, you can simply give the name of the file as the URL. To link to a section in your document, you can leave out everything in the address before the "#" symbol. For more information, check out this helpful page from SFU.

Task 2: Embedded Media

Since HTML is a plain-text format, you can't directly embed a binary image into it like you would in MS Word. Instead, you'll need to link the image into the document, similar to how you "live linked" an Excel sheet into your Word document.

  1. Find and download a reasonably large image off of the internet. Make sure you have the right to use this image in your work.

    File size considerations on the web

    Large images and video can have very large file sizes. When working with these kinds of resources on your own computer this isn't a problem, but on the Internet, the size of your resources can greatly impact the experience of using your site. Larger files take more time for you browser to download, and cost your readers time and money! In general, its a good idea to keep your HTML under 100 KB in size, keep the size of included images and stylesheets to a minimum (1 MB is a good upper limit), and never load videos automatically.

    Please keep the file size of your image for this lab under 500 KB.

    Images

    img is a "self-closing" tag used for embedding images. It looks like this:

    <img src="kitty.png" alt="A kitteh">

    Note that this element does not get a closing tag, since it does not need to contain any text in the document.

    The src attribute defines the location of the image using a URL. The text in the alt attribute is displayed in non-graphical browsers or if the image fails to load, so make it descriptive.

Task 3: Style

By now you should have a web page that looks like a very basic document. Now that it is well structured in HTML, you can apply styles to it to change the appearance. This is done by linking one or more CSS files to your document.

CSS and HTML are completely separate languages, although in practice they are almost always used together. HTML provides structured content and semantics, while CSS controls the presentation of structured content. We won't dive into how to craft your own CSS in this course, but we encourage you to look at the code for the stylesheets we've provided and try to make some small changes! CSS is a plain-text format which can be opened and edited in Notepad++ alongside your HTML.

  1. Add a reference to the CSS file at https://telic.github.io/comp152/lab/template/basic.css in the head section of the html document using the link element.

    Linking Style sheets

    The link element is a self-closing tag used to reference external resources to the html document. It must be placed inside the head section of the document (near the top).

    For linking a style sheet, there are two attributes that must be included:

    <link rel="stylesheet" href="style.css">

    The rel attribute describes the relationship of the linked file. In this case, it will be a style sheet for this document. The href attribute, just like anchor links, defines the location of the file as a URL.

    CSS Basics

    CSS works by "binding" a set of style properties to a set of structural elements. When a stylesheet is linked to an HTML document, any element in the HTML that matches a bound set in the CSS gets the associated style properties applied to it. For example, consider the following line of CSS code:

    p { color: red; }

    This line creates a binding (also known as a rule) between the set of structural elements called p and the single color style property (which changes the color of the text). As such, text inside any p elements will be colored red. The text before the first { is called a selector: it "selects" what elements will be included in the binding. Between the { and } you may include any number of property: value; pairs to define the style properties that will be set.

    Navigating FileZilla

    The main interface of FileZilla is divided into two columns. On the left, it shows the files on your local machine, with the folder hierarchy on top, and the contents of the selected folder below. On the right is the file structure on the server.

    To copy files or folders from your local system to the server and vise-versa, first navigate to the folder of the item(s) you want to copy, and to the desired destination folder on the target machine. Then, simply drag the files/folders from one pane to the other. If there is a naming conflict (the transferred file already exists on the target machine), a pop-up window will appear with options for resolving the conflict.

    Note that double-clicking files in FileZilla doesn't open them as usual. Instead, double-clicking will transfer the file as if it were dragged to the opposite pane.

    You can create a new directory by right-clicking on the containing folder in the hierarchy, and selecting create directory. You can also open the containing folder and right click in the empty space in its content view.
